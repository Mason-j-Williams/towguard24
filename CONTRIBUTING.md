Contributing Guidelines
Thank you for your interest in contributing to TowGuard24.
We welcome improvements, bug fixes, documentation updates, and new ideas that strengthen the platform.

These guidelines ensure contributions remain clean, secure, and consistent across the project.

1. How to Contribute
You can contribute in several ways:

Reporting bugs

Suggesting new features

Improving documentation

Submitting pull requests

Enhancing security or performance

Reviewing open issues

Before contributing, please review open issues to avoid duplication.

2. Submitting Issues
When opening an issue, include:

Clear description of the problem or suggestion

Steps to reproduce (if applicable)

Expected vs. actual behavior
Screenshots or logs (if helpful)

Environment details (browser, OS, device)

Be concise and respectful.

3. Pull Request Guidelines
To submit a pull request:

Fork the repository
Create a new branch for your change

Make your updates

Ensure your code is clean and secure

Write meaningful commit messages

Submit a pull request with a clear explanation

Pull requests should be focused and limited to a single purpose.

4. Coding Standards
TowGuard24 follows these principles:
Write clean, readable, maintainable code

Use consistent naming conventions

Avoid unnecessary complexity

Follow existing project structure

Include comments where clarity is needed

Never commit secrets, API keys, or credentials

Security and clarity come first.
